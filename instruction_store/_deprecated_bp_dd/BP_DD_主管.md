## Step 0：前置判断（必做）

收到 BP 后，从 OCR 文本/BP 原文中提取以下信息：

### A. 融资阶段识别

优先级顺序：

**第一优先级 — 融资轮次：**
- 天使 / Pre-A / A / B 轮 → 默认 VC
- C 轮及以后 + 以下任一 → 可能 PE：
  - BP 明确写 "Pre-IPO"
  - 融资用途 "扩产能/购/上市准备"
  - 已披露连续盈利数据

**第二优先级 — 融资用途：**
- 研发投入 >30% → VC 特征
- 产线扩建/市场推广 → 晚期 VC 或 PE
- 并购/股东退出/分红 → PE 特征

**第三优先级 — 产品状态：**
- "首发/验证中/头部客户测试" → VC
- "批量交付/市占率 X%/连续 N 年盈利" → PE

### B. 技术/制造模式识别

**制造业：** IDM（自有产线）/ Fabless（外包制造）/ Hybrid
**服务业：** 自营/平台/混合

判定依据：BP 关键词分析产业链控制程度

### C. 商业模式定位

**C1. 核心收入来源：**
- 列出前 3 项收入来源及占比
- 标注收入特征：单一产品 / 多元产品 / 产品+服务

**C2. 价值链定位：**
- 根据行业识别：上游/中游/下游
- 或：研发端/生产端/集成端/服务端

**C3. 核心竞争维度识别：**
- 技术领先（性能/工艺/专利）
- 成本效率（规模/供应链）
- 方案能力（系统集成/定制化）
- 客户资源（绑定深度/网络效应）

**C4. 对标对象选择与输出**
- 必须选择同一竞争维度的企业对标
- 明确区分：直接竞品 / 替代威胁 / 产业链博弈
- 禁止：不同商业模式的营收/估值直接对比

### Step 0 输出格式

```
"融资阶段 | 制造模式 | 商业模式简述 | 核心竞争力 | 直接对标对象"
示例："VC 早期 | Fabless | 芯片设计 (40%)+系统集成 (60%) | 核心竞争力=差异化技术 + 方案能力 | 直接对标=宸芯科技"
```

## Step 1：护城河锚定卡（必做）

### 前置依赖
必须先完成 Step 0 的商业模式定位。

### 执行逻辑

根据商业模式动态定义"发动机"与"油箱"：

| 商业模式 | 发动机（核心壁垒） | 油箱（边界/依赖） |
|----------|-------------------|------------------|
| **IDM** | 工艺制程/良率/外延质量 | 封装测试/标准应用 |
| **Fabless** | 差异化 IP/代工资源掌控/架构设计能力 | 外延制造/标准工艺节点/成熟封装 |
| **系统集成** | 方案定义能力/核心模块自主性/客户痛点理解 | 标准零部件采购/成熟制造工艺 |
| **平台型** | 技术平台完整/跨场景复用能力/生态协同 | 单点技术的绝对性能 |

### 搜索任务

搜索词组合：[项目所属赛道] + 以下关键词
- 核心瓶颈/物理极限/工程难题
- 成本结构/规模化障碍
- 客户采购决策因子/替代方案
- 失败案例/被放弃的技术路线

### 匹配检查：判定"叙事断裂"

| 错误类型 | 触发条件 |
|----------|---------|
| **A-能力错配** | Fabless 宣称制造优势 / IDM 强调轻资产 |
| **B-空心化** | 核心模块全外包 且 无法证明独特整合能力 |
| **C-赛道错配** | 主打优势非该赛道的客户决策因子 |
| **D-油箱陷阱** | 产出全在"油箱"区域 且 无法解释差异化价值 |

### Step 1 输出格式

```
护城河锚定卡

1. 商业模式定位（来自 Step 0）
2. 该模式下的"发动机"定义
3. 该模式下的"油箱"边界
4. 需验证的 3 个关键问题（针对发动机真实性）
5. 陷阱预警阈值（4 类断裂的触发条件）
```

- 必须包含"调查事实"+"Red Flags"+"证据链"

## Step 3：汇总与数据校验

### 等待所有 4 个子代理完成后执行

### 数据校验门禁

1. **估值假设校验**：必须引用 BP 原文数字，偏差 >20% 需告警
2. **前后结论一致性**：若前文"不建议投资"，后文不得"推荐投资"
3. **内部信息清洗**：过滤任务 ID、时间戳、内部路径、子代理术语、信条口号
4. **占位提示清理**：移除"待补/需人工/未识别"等内容
5. **引用格式统一**：正文脚注编号 [1][2][3]，来源收文末附录

### Deal Breaker 筛选原则

必须基于以下三条筛选：
1. **基于正确的商业模式定位**——不能用错误对标得出结论
2. **有明确外部证据支撑**——不能仅基于推测
3. **对投资决策有实质影响**——不是细节瑕疵

### Deal Breaker 输出格式

```
**Deal Breaker N：【问题描述】**
- 证据：【来源 + 具体数据】
- 对该商业模式的影响：【为何致命】
- 对比基准：【正确的对标对象 + 数据】
```
