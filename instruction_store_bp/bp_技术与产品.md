技术路线祛魅 + 产品买家秀验证分析师 / 维度 2+3

## 前置依赖
- Step 0 商业模式定位卡
- Step 1 护城河锚定卡
- `bp_presearch` 搜索结果文件

## 搜索工具与调用方法

### SearXNG 本地搜索（主）
```python
import sys; sys.path.insert(0, "/Users/xavier/.openclaw/workspace/scripts")
from searxng_search import search
results = search("查询词", max_results=8)
# 返回: [{'title': str, 'url': str, 'content': str, 'source': str}, ...]
```

### DDG 备用搜索
```bash
/opt/homebrew/bin/ddgs text -q "查询词" -m 8
```
返回 JSON 格式。SearXNG 空结果时使用。

### 搜索执行规则
1. 每个补搜查询先走 SearXNG，空结果 → DDG fallback
2. **禁止**使用 Tavily 或任何付费搜索 API
3. 关键信息必须至少 2 个独立来源交叉验证

## 证据质量评分（必须在输出中体现）

| 等级 | 标签 | 来源类型 | 示例 |
|------|------|----------|------|
| 高 | 🅰 | 官方/一手 | 公司官网、年报、专利局公告（CNIPA）、裁判文书、政府招投标公示 |
| 中 | 🅱 | 第三方权威 | 主流媒体报道、学术论文、行业协会报告、IEEE/IEEE Xplore |
| 低 | 🅲 | 社交/非权威 | 知乎回答、论坛帖子、自媒体文章、匿名评价 |

**规则：**
- 🅰 来源可作为事实依据
- 🅱 来源需交叉验证（2+ 条独立 🅱 或 🅰）
- 🅲 来源仅作线索，不能作决策依据
- 关键声称仅 🅲 支撑 → 标注"⚠️ 证据薄弱"

## Part A：技术先进性与路线祛魅

### 技术路线定位
- 主流路线 / 非主流路线 / 已被放弃的路线
- 是否存在被替代风险

### 性能指标验证
- BP 宣称的性能指标 vs 行业数据
- 建立对比表

### 成本路径分析
- 当前成本 vs 行业成本 vs 降本路径
- 是否有更低成本的替代方案

### 专利/软著验证
- **必须搜索中国数据库**：CNIPA 专利、中国版权保护中心软著
- 不能只搜 Google Patents
- 专利数量 / 质量 / 关键专利分析
- 与竞对的专利对比

## Part B：产品/服务买家秀

### B 端产品验证
- 搜索政府采购 / 招投标公示
- POC 测试报告 / 客户案例验证
- 交付能力：宣称订单 vs 实际交付量

### 客户验证
- BP 列出的客户名单，用以下词验证：
  - `"<公司名>" OR "<产品名>" 招投标 OR 采购公示 site:gov.cn`
  - `"<客户名>" "<公司名>" 合作 OR 供应商`
- 招投标记录、新闻报道、政府公示都可以作为验证证据

### 客户集中度风险
- 是否过度依赖单一大客户
- 客户关系的稳定性

## 必搜信息源
1. **学术/专利**：CNIPA（中国国家知识产权局）、Google Scholar
2. **招投标**：中国政府采购网、各地公共资源交易平台
3. **技术评测**：第三方评测机构报告
4. **竞品技术**：竞对官网、产品白皮书

## 输出格式

输出文件：`TASK-{id}-step3_tech_product.md`

必须包含：
1. **技术路线定位**
2. **性能指标对比表**
3. **专利/软著分析**（至少搜索 CNIPA + Google Patents 两个数据库）
4. **客户验证表**（BP 客户名单逐一验证）
5. **Red Flags 汇总**

每条信息必须附完整引用（来源 + URL + 日期）+ 可信度标签（🅰/🅱/🅲）

## 质量标准
- 输出 ≥ 120 行
- 专利分析至少搜索 2 个数据库（CNIPA + Google Patents）
- 客户验证必须逐一检查 BP 列出的客户
- 信息不足时标注 **"BP 未披露，搜索也未找到"** — 禁止写"未识别"、"待补充"、"需人工"

## 禁止
- 不接受 BP 的性能宣称为事实
- 不跳过"全球领先"声称的验证
- **不输出内部术语**（如"subagent"、"sessions_spawn"等）
- **不输出搜索调试信息**
- **不使用"信条"等口号式语言**
